## 项目一句话定位

**心芽** — 面向个人的私有化心得记录应用，四页面架构（萌芽/枝叶/年轮/根系），支持富文本记录、标签组织、数据统计、AI 洞察、截图分享、间隔复习（拾遗）。

- 域名：https://shuxiangnote.top
- 代码托管：GitHub + Gitee 双平台
- 服务器：阿里云 ECS（47.100.106.213）
- 数据库：PostgreSQL 13（服务器本机）

## 开发工作流

```
本地写代码 → git push origin main（GitHub）→ git push gitee main（Gitee）
→ 服务器 git fetch gitee && git reset --hard gitee/main → rm -rf .next → npm run build → pm2 restart
```

- **本地**：只写代码和调试，`npm run dev` 启动开发服务器
- **GitHub**：代码主仓库，`kangkang919/xinya`
- **Gitee**：服务器从 Gitee 拉取（国内访问稳定）
- **服务器**：`/www/wwwroot/xinya`，PM2 管理进程，SSH 用户 `admin`

## 关键约束（来自 dev-framework 附录 A-0）

1. **开发前必须核对 dev-framework 文档**，有冲突需用户确认
2. **问题诊断先判断来源**：历史遗留 vs 迭代引入，迭代问题必须 git diff 对比
3. **禁止在 `app/page.tsx` 使用无条件 redirect()**，会覆盖路由组
4. **禁止盲目叠加修复**，每次只改一个地方
5. **部署时必须清理 `.next` 目录**（`rm -rf .next`），否则 chunk 哈希冲突
6. **部署时必须主动提供服务器部署指令**
