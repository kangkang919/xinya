# 心芽各页面标题行高对齐规范

> 本文档定义多页面应用中，各页面标题区域保持视觉对齐的规范。
> 适用于所有有多个主页面、需要在页面切换时保持标题位置一致的程序。
> 新程序可参考本文档的对齐方案，根据自身 UI 设计调整具体参数。

---

## 一、问题描述

当用户在多个页面之间切换时（如底部 Tab 导航），如果各页面的标题位置不一致，会产生"跳动"感，影响体验。

**反面案例**：
- 页面 A 的标题距离顶部 20px
- 页面 B 的标题距离顶部 32px（因为有搜索框等额外元素）
- 切换时标题上下跳动

---

## 二、对齐方案

### 2.1 统一标题结构

所有主页面使用相同的标题 HTML 结构：

```tsx
{/* 页面标题 */}
<div className="flex items-center justify-between mb-1">
  <h1 className="text-xl font-bold" style={{ color: titleColor }}>
    <span style={{ color: "#8BC34A", display: "inline-block", width: "1.4em", textAlign: "center" }}>
      🌱  {/* emoji 图标 */}
    </span>
    页面名称
  </h1>
  {/* 右侧可选元素，如筛选按钮、统计数字等 */}
</div>

{/* slogan 副标题 */}
<p className="text-xs mb-5" style={{ color: dimColor }}>
  诗意化的页面描述文案
</p>
```

### 2.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 标题字号 | `text-xl`（20px） | 统一字号 |
| 标题字重 | `font-bold` | 统一粗体 |
| 标题下间距 | `mb-1`（4px） | 标题与 slogan 的间距 |
| slogan 字号 | `text-xs`（12px） | 统一小字 |
| slogan 下间距 | `mb-5`（20px） | slogan 与下方内容的间距 |
| 容器上内边距 | `px-4`（16px） | 左右内边距统一 |

### 2.3 emoji 图标对齐技巧

标题中的 emoji 图标使用固定宽度容器，确保不同 emoji 宽度不一致时标题文字仍然对齐：

```tsx
<span style={{
  color: "#8BC34A",
  display: "inline-block",
  width: "1.4em",        // ← 关键：固定宽度
  textAlign: "center"    // ← emoji 居中
}}>
  🌱  {/* 不同页面的 emoji 不同，但占用空间相同 */}
</span>
```

**为什么需要这个技巧？**
- 不同 emoji 的实际渲染宽度不同（🌱 比 🌀 窄）
- 如果不固定宽度，标题文字"萌芽"和"年轮"的起始位置会不同
- 固定 `width: 1.4em` 后，无论什么 emoji，标题文字都从同一位置开始

### 2.4 右侧元素占位

当某些页面标题右侧有按钮（如筛选、统计），而其他页面没有时，需要用占位元素保持标题居中/左对齐：

**方案 A：flex justify-between + 空占位**
```tsx
<div className="flex items-center justify-between mb-1">
  <h1>...</h1>
  <div style={{ width: "40px" }} />  {/* 占位，宽度与对面按钮一致 */}
</div>
```

**方案 B：右侧固定区域**
```tsx
<div className="flex items-center justify-between mb-1">
  <h1>...</h1>
  <div className="flex items-center gap-2">
    {hasRightElement ? <RightButton /> : <div style={{ width: "32px" }} />}
  </div>
</div>
```

## 四、完整页面模板

```tsx
export default function SomePage() {
  return (
    <div className="p-4 max-w-lg mx-auto pb-24">
      {/* ===== 标题区域（所有页面统一此结构） ===== */}
      <div className="flex items-center justify-between mb-1">
        <h1 className="text-xl font-bold" style={{ color: titleColor }}>
          <span style={{
            color: "#8BC34A",
            display: "inline-block",
            width: "1.4em",
            textAlign: "center"
          }}>
            🌱
          </span>
          页面名
        </h1>
        {/* 右侧元素或占位 */}
      </div>
      <p className="text-xs mb-5" style={{ color: dimColor }}>
        诗意化的 slogan 文案
      </p>

      {/* ===== 页面特有内容 ===== */}
      {/* ... */}
    </div>
  )
}
```

## 五、新程序适配指南

### 5.1 必须保持一致的参数

以下参数在所有主页面之间必须保持一致，否则切换时会产生跳动：

1. **标题容器的上内边距**（`p-4` 的 `pt` 部分）
2. **标题的字号和字重**（`text-xl font-bold`）
3. **标题与 slogan 的间距**（`mb-1`）
4. **slogan 的字号**（`text-xs`）
5. **slogan 与下方内容的间距**（`mb-5`）

### 5.2 可以自定义的参数

以下参数新程序可以根据设计自由调整：

| 参数 | 说明 |
|------|------|
| **标题文字颜色** | 根据主题/暗色模式调整 |
| **slogan 文字颜色** | 根据主题调整 |
| **emoji 图标颜色** | 使用品牌色或主题色 |
| **emoji 图标本身** | 使用任何 icon 或 emoji |
| **icon 的尺寸和占位宽度** | 根据 icon 大小调整 `width` 值 |
| **slogan 文案内容** | 根据产品定位编写 |
| **右侧元素** | 根据功能需求添加按钮、数字等 |

### 5.3 如果标题中有 icon

新程序如果使用 icon 组件（如 lucide-react）而非 emoji：

```tsx
<h1 className="text-xl font-bold" style={{ color: titleColor }}>
  <Sprout size={20} color="#8BC34A" style={{ display: "inline-block", width: "1.4em", textAlign: "center", verticalAlign: "middle" }} />
  萌芽
</h1>
```

关键：icon 的外层容器仍然需要 `display: inline-block; width: 1.4em; textAlign: center` 来保证对齐。icon 的具体尺寸、颜色、样式遵循新程序的设计要求。

### 5.4 验证方法

开发完成后，快速切换各页面，观察：
1. 标题文字是否在同一水平线上？
2. slogan 是否在同一水平线上？
3. 切换时是否有上下跳动？

如果有跳动，检查各页面的 `mb-1`、`mb-5`、`p-4` 等间距值是否完全一致。
