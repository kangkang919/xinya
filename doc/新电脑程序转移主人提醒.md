## 四、环境变量配置

项目需要 `.env` 文件（本地开发）或 `.env.production`（生产部署）。

从 `D:\Project Qoder\xinya\.env.example` 复制一份，填写真实值：

```bash

cp .env.example .env

# 需要填写的真实值：
DATABASE_URL="postgresql://xinya:你的密码@localhost:5432/xinya_db"
JWT_SECRET="随机字符串，32位以上"
SMTP_USER="1243177461@qq.com"
SMTP_PASS="你的SMTP授权码"
NEXT_PUBLIC_BASE_URL="http://localhost:3000"  # 本地开发用这个
```

️ **`.env` 文件包含敏感信息，不要上传到 GitHub！**（已在 .gitignore 中排除）

## 五、本地开发验证

## 六、部署到服务器（从 Gitee 拉取）

```bash
# 在服务器上执行：
cd /www/wwwroot/xinya
git fetch gitee
git reset --hard gitee/main
npm run build
pm2 restart xinya
```

服务器端已配置好 gitee 远程仓库，从 Gitee 拉取代码（GitHub 在国内访问不稳定）。

## 七、重要提醒

1. **`.env.production` 文件**包含真实密码和密钥，务必妥善保管，不要泄露。已备份到 `doc/.env.production.备份`
2. **GitHub PAT 令牌**有有效期，过期后需要重新生成
3. **QQ 邮箱 SMTP 授权码**如果长时间不用可能失效，需要重新生成
4. **数据库密码**如果忘记，需要在服务器上重置 PostgreSQL 用户密码
5. **域名 SSL 证书**由宝塔面板自动管理（Let's Encrypt），一般不需要手动操作
6. **PM2 日志**：`pm2 logs xinya` 查看运行日志，排查问题用
7. **PowerShell 路径含空格**：`cd 'D:\Project Qoder\xinya'` 必须用引号包裹路径
8. **双平台推送**：每次本地修改后需分别推送到 GitHub 和 Gitee（`git push origin main` + `git push gitee main`）
