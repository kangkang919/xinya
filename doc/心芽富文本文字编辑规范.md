# 心芽富文本文字编辑规范

> 本文档定义富文本编辑器的功能边界和实现规范。
> 适用于所有需要富文本编辑功能的程序项目。
> 新程序可参考本文档的功能取舍逻辑，根据自身需求调整。

---

## 一、功能清单（做什么 & 不做什么）

### ✅ 必须实现的功能

| 功能 | 说明 | 实现方式 |
|------|------|---------|
| **加粗** | 选中文字后切换加粗 | `document.execCommand("bold")` |
| **斜体** | 选中文字后切换斜体 | `document.execCommand("italic")` |
| **无序列表** | 创建/取消 bullet list | `document.execCommand("insertUnorderedList")` |
| **文字颜色** | 从预设色板中选择文字颜色 | `document.execCommand("foreColor", color)` |
| **字数统计** | 实时显示纯文字字数（不含 HTML 标签和空格） | `content.replace(/<[^>]*>/g, "").replace(/\s/g, "").length` |

### ❌ 明确不实现的功能

| 功能 | 不实现的原因 |
|------|-------------|
| **标题功能（H1/H2/H3）** | 心得是个人随笔，不需要文档式层级标题。标题由独立的标题输入框处理，正文中不需要再嵌套标题 |
| **字体大小选择** | 保持全文统一字号，避免排版混乱。个人笔记不需要大小标题的视觉层级 |
| **字体选择（字体族）** | 统一使用系统默认字体，保证跨平台一致性。**新程序如需字体选择功能，遵循新程序的 UI 设计要求** |
| **文字对齐（左/中/右）** | 笔记默认左对齐即可，不需要居中或右对齐 |
| **下划线** | 与链接样式冲突，且个人笔记中很少需要 |
| **删除线** | 使用频率极低，保持工具栏简洁 |
| **有序列表** | 无序列表已够用，有序列表增加复杂度 |
| **引用块** | 个人笔记场景不需要 |
| **代码块** | 个人笔记场景不需要 |
| **超链接** | 个人笔记场景不需要 |
| **撤销/重做按钮** | 浏览器原生 Ctrl+Z / Ctrl+Y 已满足需求 |

### ️ 遵循新程序要求的功能

| 功能 | 说明 |
|------|------|
| **图片上传/插入** | 本程序不实现图片功能。新程序如需图片功能，遵循新程序的产品设计和技术方案 |
| **字体选择** | 本程序统一使用系统字体。新程序如需字体选择器，遵循新程序的 UI 设计要求 |
| **表格** | 本程序不实现。新程序按需决定 |

## 二、工具栏布局规范

### 2.1 工具栏结构

```
[加粗] [斜体] [列表] [颜色▼] | [标签] [专注模式]          [字数]
                              ↑ 分隔线                      ↑ 右对齐
```

### 2.2 按钮规格

| 属性 | 值 | 说明 |
|------|-----|------|
| 按钮尺寸 | 32×32px（p-2 + icon 18px） | 触摸友好，不占过多空间 |
| 图标颜色 | `#666`（默认） | 中性灰，不抢视觉焦点 |
| 按钮圆角 | `rounded-lg`（8px） | 柔和圆角 |
| hover 效果 | `hover:bg-gray-100` | 浅灰背景反馈 |
| 分隔线 | 1px 宽，`#e0e0e0`，高 20px | 视觉分组 |

### 2.3 工具栏位置

- 位于标题输入框下方、正文编辑区上方
- 使用 `border-b` 底部分隔线
- 横向排列，超出时 `overflow-x-auto` 可横向滚动（移动端适配）

## 三、文字颜色规范

### 3.1 预设色板（6 色）

| 颜色 | Hex | 用途建议 |
|------|-----|---------|
| 深灰 | `#333333` | 默认正文色，恢复默认 |
| 嫩绿 | `#8BC34A` | 强调、关键词 |
| 天蓝 | `#42A5F5` | 补充说明 |
| 橙色 | `#FF8C42` | 警示、重要 |
| 棕色 | `#795548` | 引用、备注 |
| 红色 | `#e57373` | 错误、否定 |

### 3.2 颜色选择器交互

- 点击调色板按钮展开色板
- 色板为横向排列的圆形色块（7×7 圆，28×28px）
- 选中颜色后自动收起色板
- 色板使用绝对定位（`absolute top-10 left-0`），z-index 确保在最上层

### 3.3 新程序颜色建议

新程序可根据品牌色调整预设色板，但建议：
- 保留一个"恢复默认"色（通常是正文色）
- 颜色数量控制在 4-8 个，避免选择困难
- 颜色之间要有足够的区分度

## 四、编辑器核心实现

### 4.1 技术方案

使用 `contentEditable` + `document.execCommand` 方案：

```tsx
const editorRef = useRef<HTMLDivElement>(null)

function execCmd(cmd: string, value?: string) {
  document.execCommand(cmd, false, value)
  editorRef.current?.focus()
  updateContent()
}

function updateContent() {
  if (editorRef.current) setContent(editorRef.current.innerHTML)
}
```

### 4.2 编辑器容器样式

```tsx
<div
  ref={editorRef}
  contentEditable
  className="w-full outline-none text-sm leading-relaxed"
  style={{
    padding: "16px",
    minHeight: "30vh",
    color: "#333"
  }}
  onInput={updateContent}
  dangerouslySetInnerHTML={{ __html: content }}
  data-placeholder="在这里写下你的感悟、想法或日记…"
/>
```

关键样式说明：
- `outline-none`：去掉聚焦时的蓝色边框
- `text-sm`：正文 14px（Tailwind 默认）
- `leading-relaxed`：行高 1.625，舒适阅读
- `minHeight: 30vh`：最小高度，避免编辑器太矮
- `data-placeholder`：配合 CSS 实现空状态提示文字

### 4.3 标题输入框

标题是独立的 `<input>` 元素，不在 contentEditable 内：

```tsx
<input
  className="w-full text-xl font-bold outline-none px-4 pt-6 pb-2"
  style={{ color: "#333", background: "transparent" }}
  placeholder="给这颗种子取个名字…"
  value={title}
  onChange={e => setTitle(e.target.value)}
/>
```

- 标题与正文分离，标题不支持富文本格式
- `text-xl font-bold`：20px 粗体
- `background: transparent`：透明背景，融入页面

### 4.4 列表样式补充

`contentEditable` 生成的 `<ul>` 和 `<ol>` 默认没有样式，需要手动添加 CSS：

```css
.view-content ul { list-style: disc; padding-left: 1.5em; margin: 0.5em 0; }
.view-content ol { list-style: decimal; padding-left: 1.5em; margin: 0.5em 0; }
.view-content li { margin: 0.2em 0; }
```

️ **这是常见坑**：`execCommand("insertUnorderedList")` 执行成功了，但因为没有 CSS 样式，用户看不到 bullet 效果，误以为功能没生效。

### 4.5 光标恢复

执行 `execCommand` 后编辑器会失去焦点，必须手动恢复：

```tsx
function execCmd(cmd: string, value?: string) {
  document.execCommand(cmd, false, value)
  editorRef.current?.focus()  // ← 必须加这行
  updateContent()
}
```

## 五、常见问题与避坑

### 5.1 execCommand 执行了但看不到效果

**原因**：CSS 样式缺失。`execCommand` 只生成 HTML 标签（如 `<ul>`），但不添加 CSS 样式。

**解决**：为 `ul`、`ol`、`li` 添加 CSS 样式（见 4.4 节）。

### 5.2 点击工具栏按钮后光标位置丢失

**原因**：点击按钮导致编辑器失去焦点，选区消失。

**解决**：在 `execCmd` 中调用 `editorRef.current?.focus()` 恢复焦点（见 4.5 节）。

### 5.3 粘贴外部内容格式混乱

**原因**：从网页/Word 粘贴会带入大量 HTML 样式。

**解决**：监听 `paste` 事件，用 `e.clipboardData.getData('text/plain')` 获取纯文本，再用 `execCommand("insertText", text)` 插入。

### 5.4 execCommand 已被标记为废弃

`document.execCommand` 在 MDN 上标记为 deprecated，但目前所有主流浏览器仍支持。替代方案是使用 Selection API + Range API 手动操作 DOM，但复杂度显著增加。对于简单的富文本需求，`execCommand` 仍然是最实用的方案。

## 六、新程序适配指南

如果你的新程序需要富文本编辑功能，按以下步骤决策：

1. **确定功能范围**：参考第一节的功能清单，根据产品需求勾选需要的功能
2. **选择技术方案**：简单需求用 `contentEditable + execCommand`，复杂需求考虑 Slate.js / TipTap / Lexical
3. **设计工具栏**：参考第二节的布局规范，按功能数量调整
4. **定义颜色体系**：参考第三节，根据品牌色调整预设色板
5. **补充 CSS 样式**：确保列表、加粗、斜体等有正确的视觉表现
6. **处理边界情况**：光标恢复、粘贴格式、空状态提示
