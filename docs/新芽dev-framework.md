## 一、产品定位

| 项目 | 内容 |
| :--- | :--- |
| **名称** | 心芽 |
| **Slogan** | 记录内心的每一次萌发 |
| **核心价值** | 私有记录 + 可控分享 |
| **技术方案** | Next.js 14 App Router + TypeScript + Tailwind CSS + Prisma + PostgreSQL；响应式网页，PC 与手机端统一编辑体验；支持 PWA「添加到主屏幕」 |
| **双端策略** | 网页端专注写作编辑，手机端便捷回顾学习；手机端与网页端富文本编辑功能完全一致 |
| **代码管理** | GitHub 仓库 + PAT 认证；本地修改后通过 GitHub REST API 推送，服务器执行 `git pull && npm run build && pm2 restart xinya` 部署 |

## 三、心得数据模型

| 字段 | 规则 |
| :--- | :--- |
| 标题 | 必填，简短 |
| 正文 | 富文本编辑器（见第四章） |
| 标签 | **至少一个，强制存在**；纯文字，≤8字；支持多标签；未选择或全部删除时自动使用默认标签"随笔" |
| 心情标记 | 可选，5个选项：开心 😊、难过 😢、平静 😐、激动 🤩、忧虑 😟 |
| 记录时间 | 自动生成 + 可手动修改；**所有时间统计按北京时间（Asia/Shanghai）计算** |
| 收藏状态 | 可选，默认未收藏；卡片右上角书签图标操作 |
| 置顶状态 | 允许多个心得同时置顶，不限制数量 |

**已删除字段/功能：**
- 配图（图片功能整体删除）
- `isDraft` 草稿状态（见第四章变更）

**标签强制存在规则：**
1. 新建/编辑心得时若未选择任何标签，保存后自动打上默认标签"随笔"。
2. 删除某个标签时，仅含该标签的心得自动补上"随笔"；含多个标签的心得仅移除被删标签。

## 四、富文本编辑器规范

### 4.1 通用规则（手机端与网页端完全一致）

| 项目 | 规则 |
| :--- | :--- |
| 默认字体 | 微软雅黑 |
| 默认字号 | 12号 |
| 字体颜色 | 仅四色可选——红、黄、蓝、默认黑色 |
| 无标题层级 | 不设H1/H2/H3等标题选择 |
| 粘贴净化 | 所有外部粘贴文本统一强制转化为上述格式；不支持图片粘贴或上传 |
| 支持功能 | 加粗、斜体、有序/无序列表、撤销/重做 |
| 字数统计 | 实时显示在编辑器右下角，浅灰色 |
| 手机端工具栏 | 编辑心得时顶部格式化工具栏始终固定在顶部（fixed 定位），滚动不消失 |

### 4.2 网页端专属增强

- 专注模式（全屏编辑，按ESC或左上角按钮退出）

### 4.3 保存与草稿（已变更）

**v2.0 原规则：** 每30秒自动保存草稿；离线时保存为草稿；存在"存草稿"按钮。

**v2.1 更新规则：**
- **彻底移除"存草稿"功能与按钮**。
- 所有保存入口统一为直接保存正式心得，`isDraft` 固定为 `false`。
- 离线保存提示文案保留，但实际行为待确认（当前实现为前端提示，无本地持久化草稿）。

### 4.4 编辑态按钮文案

| 场景 | 文案 |
| :--- | :--- |
| 新建按钮 | 心芽，记录内心的每一次萌发 |
| 编辑按钮 | 续叶，重温这片心得 |

| 多设备登录 | 允许多设备同时登录，不互相踢下线 |

### 5.2 萌芽页

**页面结构（从上到下）：**
1. 顶部固定导航栏（页面标题 + 布局切换 + 退出按钮）
2. 搜索栏 + 收藏筛选按钮（⭐）+ 时间范围快捷筛选（最近7天 / 30天 / 3个月 / 全部）
3. 今日速览卡片（随内容一起滚动）
4. 心得卡片列表

**今日速览卡片规则：**
- 展示：今日心得数、本周心得数、最近一条心得预览、连续记录天数、历史最长连续天数
- 今日已记录≥1条后，自动折叠为一行小字"今日已播种 🌱"，点击可展开
- **所有统计按北京时间计算；周一为一周起点**
- 连续记录天数算法：从今天或昨天开始往前连续有记录的天数；断开则当前连续归零
- 历史最长连续：历次连续段中的最大值

**卡片展示规则：**

| 项目 | 规则 |
| :--- | :--- |
| 布局 | 默认卡片流，支持切换时间轴布局 |
| 卡片内容 | 标题、首行预览（2行截断）、标签胶囊、记录时间、心情图标 |
| 收藏图标 | 卡片右上角书签形状；未收藏灰色空心，已收藏嫩绿色实心；点击触发弹跳动画 |
| 长按操作 | 编辑、删除、置顶 |
| 置顶规则 | **多个心得可同时置顶，不限制数量** |
| 收藏规则 | 多篇可同时收藏 |
| 收藏筛选 | 搜索栏旁⭐按钮，点击只显示收藏心得，再点取消 |
| 新建引导文案 | "此刻思绪万千，别忘了为它贴上专属的叶片标签 🌿" |

**删除操作弹窗文案：**

> **"确定要让这片叶子飘落吗？"**
> 一旦飘落，便无法追回。
>
> 【让它飘落】　　　【再想想】

确认删除后直接删除，无回收站。

## 七、新用户引导（播种引导）

**触发时机**：首次注册成功后自动进入

**三步流程：**
1. 欢迎插画页（嫩芽破土）+ 欢迎语：「欢迎来到心芽，这里是你内心的花园 🌱」
2. 创建第一个标签引导
3. 写下第一条心得引导

**每步都有"跳过，直接进入"小字按钮**，点击后直接进入主界面。

## 十五、开发执行原则

1. **唯一规范源**：本文档是心芽程序的最高执行标准。任何后续开发必须首先回顾本文档，确认无冲突、不超出范围。
2. **冲突/超范围处理**：若新需求或实现方案与本文档冲突，或超出当前已确认范围，必须向用户明确说明冲突点，征得用户明确同意后方可执行。
3. **文档同步**：每次因新需求修改本文档所涉规则后，必须同步更新本文件，保持其时效性与权威性。
4. **分批推送、分批验收**：每个任务独立完成、独立推送、独立验收，确认通过后再进行下一项。
5. **时区统一**：所有时间相关计算统一按北京时间（Asia/Shanghai）处理。
