## 简介
本指南面向首次接触"心芽"项目的开发者，提供从环境准备、本地开发到生产部署的完整路径。你将了解 Node.js 版本要求、PostgreSQL 配置、第三方服务（邮箱 SMTP、AI API）密钥设置，以及常用脚本与常见问题处理方案。特别针对Windows 10用户提供了详细的兼容性指导和配置优化建议。

## 环境准备清单

### 系统要求
- **操作系统**: Windows 10/11, macOS, Linux
- **Node.js**: 推荐 v18.x 或更高版本（支持 LTS 版本）
- **数据库**: PostgreSQL 12+ 
- **包管理器**: npm 或 yarn

### 必需软件安装

#### Node.js 安装与验证
```bash
# 检查当前Node.js版本
node -v

# 推荐使用nvm进行版本管理（跨平台）
# Windows: https://github.com/coreybutler/nvm-windows
# macOS/Linux: https://github.com/nvm-sh/nvm

# 安装指定版本
nvm install 18
nvm use 18
```

#### PostgreSQL 数据库配置
```bash
# Windows: 使用安装包或Docker
# Docker方式（推荐）
docker run --name xinya-postgres -e POSTGRES_PASSWORD=your_password -p 5432:5432 -d postgres:15

# 创建数据库和用户
psql -U postgres -h localhost -p 5432
CREATE DATABASE xinya;
CREATE USER xinya WITH PASSWORD 'xinya_password';
GRANT ALL PRIVILEGES ON DATABASE xinya TO xinya;
```

#### Git 配置（可选但推荐）
```bash
# 配置Git用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Windows用户注意：确保已安装Git并添加到PATH
```

## 本地开发环境搭建

### 项目克隆与初始化
```bash
# 克隆项目
git clone <repository-url>
cd xinya

# 安装依赖
npm install
# 或使用yarn
yarn install

# 初始化数据库
npm run db:deploy
```

### 环境变量配置
在项目根目录创建 `.env` 文件：

```env
# 数据库配置
DATABASE_URL="postgresql://xinya:xinya_password@localhost:5432/xinya"

# JWT安全密钥（生产环境请使用强随机字符串）
JWT_SECRET="your-super-secret-jwt-key-change-in-production"

# 邮箱SMTP配置（QQ邮箱示例）
SMTP_USER="your_email@qq.com"
SMTP_PASS="your_smtp_authorization_code"

# 应用URL配置
NEXT_PUBLIC_BASE_URL="http://localhost:3000"
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# AI功能配置（可选）
DEEPSEEK_API_KEY="your_deepseek_api_key"
```

### Windows 10 特殊配置

#### 权限问题处理
```powershell
# 以管理员身份运行PowerShell
# 设置执行策略
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 如果遇到权限错误，尝试以下命令
icacls . /grant Users:(OI)(CI)F /T
```

#### 路径和编码问题

```bash
# 确保使用UTF-8编码
chcp 65001

# 设置Node.js全局路径
set NODE_PATH=%APPDATA%\npm\node_modules

# 如果使用VS Code，确保终端设置为PowerShell或CMD
```

## 首次运行配置

### 邮箱服务配置
1. **QQ邮箱配置步骤**:
   - 登录QQ邮箱 → 设置 → 账户 → POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务
   - 开启SMTP服务并获取授权码
   - 将授权码填入 `SMTP_PASS` 环境变量

2. **其他邮箱提供商**:
   - Gmail: 需要启用应用专用密码
   - Outlook: 使用应用密码或OAuth2
   - 企业邮箱: 联系IT部门获取SMTP配置

### AI API设置（可选）
1. **DeepSeek API配置**:
   - 访问 DeepSeek 官网注册账号
   - 在控制台创建API Key
   - 将Key填入 `DEEPSEEK_API_KEY` 环境变量

2. **测试AI功能**:
   - 登录后进入复习功能
   - 选择条目进行AI分析
   - 检查是否成功生成题目和要点

### 基础功能验证

## 常见问题排查

### 数据库连接问题
```bash
# 检查PostgreSQL服务状态
# Windows: services.msc 查找 PostgreSQL 服务
# macOS: brew services list | grep postgres
# Linux: systemctl status postgresql

# 测试数据库连接
psql -U xinya -h localhost -p 5432 -d xinya

# 重新运行迁移
npm run db:deploy
```

### 端口冲突解决
```bash
# 检查3000端口占用
# Windows
netstat -ano | findstr :3000
taskkill /PID <进程ID> /F

# macOS/Linux
lsof -i :3000
kill -9 <进程ID>

# 修改端口号
# 编辑 package.json 中的 dev 脚本
# "dev": "next dev -p 3001"
```

### 权限和路径问题（Windows）
```powershell
# 清理npm缓存
npm cache clean --force

# 重新安装依赖
rm -rf node_modules package-lock.json
npm install

# 检查文件编码
chcp 65001
```

### 邮件发送失败
- 确认SMTP服务器地址和端口正确
- 检查授权码而非登录密码
- 验证防火墙未阻止SMTP连接
- 查看应用日志获取详细错误信息

### AI接口报错
- 检查网络连接和代理设置
- 验证API Key有效性
- 确认请求频率限制
- 查看响应格式是否符合预期

## 生产环境部署

### 服务器要求
- **操作系统**: Ubuntu 20.04+, CentOS 8+, 或等效Linux发行版
- **Node.js**: v18.x LTS
- **PostgreSQL**: 12+
- **PM2**: 进程管理器
- **Nginx**: 反向代理（可选）

### 部署步骤
```bash
# 1. 克隆项目到服务器
git clone <repository-url>
cd xinya

# 2. 安装生产依赖
npm ci --production

# 3. 构建应用
npm run build

# 4. 运行数据库迁移
npm run db:deploy

# 5. 使用PM2启动
pm2 start ecosystem.config.js

# 6. 设置开机自启
pm2 startup
pm2 save
```

### Nginx配置示例
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 环境变量配置
```bash
# 在生产服务器上创建.env.production
NODE_ENV=production

JWT_SECRET="strong-random-secret-key"

NEXT_PUBLIC_BASE_URL="https://your-domain.com"
NEXT_PUBLIC_APP_URL="https://your-domain.com"
DEEPSEEK_API_KEY="your-api-key"
```

## 附录：环境变量清单与示例

### 必需环境变量
| 变量名 | 说明 | 示例值 | 必填 |
|--------|------|--------|------|
| DATABASE_URL | PostgreSQL数据库连接串 | `postgresql://user:pass@localhost:5432/dbname` | ✅ |
| JWT_SECRET | JWT签名密钥 | `your-secret-key-here` | ✅ |
| SMTP_USER | 邮箱SMTP用户名 | `your_email@qq.com` | ✅ |
| SMTP_PASS | 邮箱SMTP授权码 | `abcdefghijklmnop` | ✅ |

### 可选环境变量
| 变量名 | 说明 | 示例值 | 默认值 |
|--------|------|--------|--------|
| NEXT_PUBLIC_BASE_URL | 前端基础URL | `http://localhost:3000` | `http://localhost:3000` |
| NEXT_PUBLIC_APP_URL | 应用对外访问URL | `http://localhost:3000` | `http://localhost:3000` |
| DEEPSEEK_API_KEY | DeepSeek API密钥 | `sk-your-key-here` | 空（禁用AI功能） |
| NODE_ENV | 运行环境 | `development` 或 `production` | `development` |

### 环境变量文件结构
```
项目根目录/
├── .env                    # 本地开发环境
├── .env.local              # 本地覆盖配置（不提交到Git）
├── .env.production         # 生产环境配置
└── .env.example            # 环境变量模板
```

### 安全注意事项
- 永远不要将包含敏感信息的.env文件提交到版本控制
- 使用强随机字符串作为JWT_SECRET
- 定期轮换SMTP授权码和API密钥
- 在生产环境中使用环境变量管理服务
