- 运行时入口：`middleware.ts` 在请求级拦截 `/api/` 与静态资源，基于 `xinya_token` Cookie 对非认证页面做登录态校验，是前后端共享的鉴权边界。
- 依赖分层：`shared_components`（React UI）与 `server_lib`（Prisma Client、认证、邮件、AI、复习调度等）通过顶层 `package.json` 的 npm 脚本 (`dev/build/start/db:deploy`) 统一构建；`postinstall` 自动执行 `prisma generate`，确保服务端模型变更后客户端类型同步。
- 路径别名：`tsconfig.json` 配置 `@/*` → `./*`，使各子模块可跨包使用绝对路径引用，避免相对路径地狱。
- 生产部署：`deploy.sh` 串联 `npm install → prisma generate → prisma migrate deploy → next build → pm2 start ecosystem.config.js`，`ecosystem.config.js` 固定以 PM2 单实例启动 `next start -p 3000`，配合 `docs` 中的环境变量备份完成上线闭环。
- 代码质量：`eslint.config.mjs` 继承 `eslint-config-next/core-web-vitals` 与 `typescript` 规则，`postcss.config.mjs` 接入 Tailwind v4，三者由根脚本 `lint` 统一触发。
