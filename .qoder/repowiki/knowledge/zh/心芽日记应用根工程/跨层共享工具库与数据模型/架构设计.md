模块按职责拆分为独立文件，通过命名函数级导入形成单向依赖链：
- 数据层：`prisma/schema.prisma` 声明 User/Entry/Tag/Share/AiInsight/InsightReport/GrowthLog/EmailToken/MagicLink/QuizQuestion/QuizRecord/UserSetting/ReviewCallLog 等实体，依赖 PostgreSQL；`lib/prisma.ts` 以 `globalForPrisma` 单例模式在 dev 下缓存全局 `PrismaClient` 实例并按环境切换日志级别。
- 认证层：`lib/auth.ts` 基于 bcryptjs + jsonwebtoken 实现密码哈希、JWT 签发/校验，并通过 Next `cookies()` API 从 `xinya_token` Cookie 提取当前用户 ID。
- 外部服务适配：`lib/mailer.ts` 封装 QQ SMTP（nodemailer, 465/SSL）发送验证码、Magic Link、重置密码三类邮件；`lib/deepseek.ts` 调用 DeepSeek Chat Completions (`deepseek-chat`) 接口生成题目与要点总结，内置 30s AbortController 超时与最大重试循环。
- 业务编排：`lib/review-scheduler.ts` 实现间隔重复（2^n 指数退避）复习调度，优先答错/久未复习记录，无题时回退到 `template-questions.ts` 的模板生成器，同时维护 ReviewCallLog 最近 30 条裁剪。
- 工具与 UI：`lib/utils.ts` 提供验证码、随机 Token、HTML 清洗、日期计算、连续打卡天数统计；`lib/export-utils.ts` 将心得导出为 Markdown 并触发浏览器下载；`lib/useTheme.ts` 作为 React Client Hook 管理 localStorage 中的主题并在多标签页间通过自定义事件同步。
依赖方向单向向外：scheduler → prisma/template-questions → deepseek，auth 仅依赖 Next headers，export-utils 与 useTheme 不依赖其他 lib 文件。
